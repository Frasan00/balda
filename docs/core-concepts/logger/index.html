Logger

Balda provides built-in structured logging powered by Pino - a fast, low-overhead logger for Node.js, Bun, and Deno.

Overview​

  • Library: Pino ^10.1.0
  • Default level: info
  • Format: JSON (structured logs)
  • Runtimes: Node.js, Bun, Deno

Basic Usage​

import { logger } from 'balda';

logger.info('Server starting');
logger.error('Something went wrong', { error: err });
logger.warn('High memory usage');
logger.debug('Debug information', { context: data });

Global Configuration​

Use defineLoggerConfig() to override the default logger configuration. This must be called before creating the server or importing services that use the logger.

import { defineLoggerConfig, Server } from 'balda';

// Configure logger first
defineLoggerConfig({
  level: 'debug',
});

const server = new Server();

Configuration Options​

Balda's defineLoggerConfig() accepts any Pino options:

OptionTypeDefaultDescription
levelstring"info"Log level (trace, debug, info, warn, error, fatal, silent)
formattersobject-Custom formatters for log entries
transportobject-Transport for pretty-printing (development)
timestampboolean | functiontrueAdd timestamp to logs
redactarray-Redact sensitive fields

Pretty-Printed Logs (Development)​

defineLoggerConfig({
  level: 'debug',
  transport: {
    target: 'pino-pretty',
    options: {
      colorize: true,
      translateTime: 'HH:MM:ss Z',
      ignore: 'pid,hostname',
    },
  },
});

JSON Logs (Production)​

defineLoggerConfig({
  level: 'warn',
  formatters: {
    level: (label) => {
      return { level: label };
    },
  },
  timestamp: pino.stdTimeFunctions.isoTime,
});

Environment-Based Configuration​

defineLoggerConfig({
  level: process.env.LOG_LEVEL || 'info',
  // Only use transport in development
  ...(process.env.NODE_ENV === 'development' && {
    transport: {
      target: 'pino-pretty',
      options: { colorize: true },
    },
  }),
});

Scoped Loggers​

Create child loggers with additional context using logger.child():

import { logger } from 'balda';

const serviceLogger = logger.child({ scope: 'UserService' });
serviceLogger.info('User created', { userId: '123' });

// Output: {"level":"info","time":...,"scope":"UserService","userId":"123","msg":"User created"}

Scoped Logger Pattern in Classes​

import { logger } from 'balda';

export class UserService {
  private readonly log = logger.child({ scope: 'UserService' });

  async createUser(email: string) {
    this.log.info('Creating user', { email });
    // ...
  }
}

Service Logger Override​

Important

Services like MqttService, QueueService, CronService, and CommandRegistry have independent static loggers created at class initialization. To configure these, you must call defineLoggerConfig() before importing these services.

Correct Order​

// 1. Configure logger FIRST
import { defineLoggerConfig } from 'balda';
defineLoggerConfig({ level: 'debug' });

// 2. Then import services
import { QueueService } from 'balda';
import { MqttService } from 'balda';

// Now these services will use your configured logger

Manual Service Logger Override​

If you need to reconfigure a service's logger after initialization:

import { logger } from 'balda';
import { QueueService } from 'balda';

// Override the service's static logger
QueueService.logger = logger.child({
  scope: 'QueueService',
  customContext: 'production',
});

Log Levels​

LevelNumeric ValueUse Case
trace10Very detailed debugging (not recommended in production)
debug20Debugging information
info30General informational messages
warn30Warning messages
error40Error messages
fatal60Critical errors that require immediate attention
silentInfinityDisable logging

API Reference​

defineLoggerConfig(options?): Logger​

Configures the global logger instance.

import { defineLoggerConfig } from 'balda';

defineLoggerConfig({
  level?: string;           // Log level (default: "info")
  formatters?: object;      // Custom formatters
  transport?: object;       // Transport configuration
  timestamp?: boolean;      // Enable/disable timestamp
  // ... any Pino option
});

Returns: The configured Pino logger instance.

note

Calling defineLoggerConfig() multiple times will replace the previous logger configuration. Services with static loggers created before the last call will continue using their original logger unless manually overridden.

logger​

The default logger instance.

import { logger } from 'balda';

logger.info(message, ...args);
logger.error(message, ...args);
logger.warn(message, ...args);
logger.debug(message, ...args);
